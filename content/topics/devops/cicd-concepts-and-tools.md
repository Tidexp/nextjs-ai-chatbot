# CI/CD Concepts and Tools

**Explore the Continuous Integration and Continuous Delivery/Deployment pipeline stages and review common tools like Jenkins, GitLab CI, and GitHub Actions.**

---

## Continuous Integration (CI)

Automatically build and test code changes:

- Commit code to repository
- Automated build triggered
- Run tests
- Report results

---

## Continuous Delivery/Deployment (CD)

**Continuous Delivery:** Code is always ready to deploy
**Continuous Deployment:** Automatically deploy to production

---

## CI/CD Pipeline Stages

1. **Source**: Code commit triggers pipeline
2. **Build**: Compile and package application
3. **Test**: Run unit, integration tests
4. **Deploy**: Release to staging/production
5. **Monitor**: Track application health

---

## Popular Tools

- **GitHub Actions**: Integrated with GitHub
- **GitLab CI/CD**: Built into GitLab
- **Jenkins**: Open-source automation server
- **CircleCI**: Cloud-based CI/CD
- **Travis CI**: GitHub integration
